# RAG Eval Harness — Sub-plan 1A: Eval Engine

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the pure-Python foundation of the RAG eval harness — schemas, model pricing, retrieval/generation/refusal/operational metrics, statistical wrappers, and dev-set loaders — fully tested in isolation. No API, no frontend, no runner, no observability.

**Architecture:** A new `src/eval/` package containing typed Pydantic schemas, four metric modules (each pure functions), bootstrap-CI and paired-permutation-test statistics, and two dataset loaders (`squad_v2`, `ml_papers`). Generation metrics reuse the existing `src/evaluation.py` LLM-as-judge functions. Every module has a single responsibility and is unit-tested with deterministic seeds.

**Tech Stack:** Python 3.10+, Pydantic v2, scikit-learn cosine, NumPy, sentence-transformers (already a Chroma dep), `datasets` (Hugging Face) for SQuAD loading, pytest.

**Spec:** [`docs/superpowers/specs/2026-04-26-rag-eval-harness-phase-1-design.md`](../specs/2026-04-26-rag-eval-harness-phase-1-design.md)

## Task 5 — `src/eval/metrics/retrieval.py`

**Files:** `tests/test_eval_metrics_retrieval.py`, `src/eval/metrics/retrieval.py`

**Functions** (operate on `Sequence[str]`, return `float`, return `NaN` when gold is empty):
- `recall_at_k(gold, retrieved, k) -> float` — `|gold ∩ retrieved[:k]| / |gold|`.
- `mrr_at_k(gold, retrieved, k) -> float` — reciprocal of first hit rank in `retrieved[:k]`, else 0.
- `ndcg_at_k(gold, retrieved, k) -> float` — binary relevance, `discount = 1 / log2(rank + 1)`.

**Test cases per metric:**
- Perfect ranking → 1.0.
- No hit → 0.0.
- Hit at exact rank N (hand-compute expected).
- Truncation behavior at k.
- Empty gold returns NaN.
- For `mrr`: multi-gold uses FIRST hit rank only.
- For `ndcg`: hand-compute with 2 gold + 1 hit at rank 2.

**Implementation notes:**
- Architecture-header docstring; `WHY rank+1 inside log2` comment for nDCG.
- `from typing import Sequence`; constant `NAN = float("nan")`.

## Task 7 — `src/eval/statistics.py`: bootstrap CI

**Files:** `tests/test_eval_statistics.py`, `src/eval/statistics.py`

**`bootstrap_ci(values, n_resamples=1000, seed=12345) -> (mean, ci_low, ci_high)`:**
- Drop NaN values before resampling.
- All-NaN input raises `ValueError`.
- Single-value input short-circuits to `(v, v, v)`.
- Use `numpy.random.default_rng(seed).integers(0, n, size=(n_resamples, n))` for resample indices, then `arr[indices].mean(axis=1)`.
- Return 2.5th and 97.5th percentile of the bootstrap means.

**Test cases:**
- 200 normal samples around mean=5: CI brackets 5.0.
- Same seed twice → identical output.
- NaN values dropped (mean of [1,2,3,4] with two NaNs).
- All-NaN raises `ValueError`.
- Single value → zero-width CI.

Commit: `feat(eval): add bootstrap_ci for percentile confidence intervals`.

## Task 8 — `src/eval/statistics.py`: paired permutation test

**Files:** `tests/test_eval_statistics.py` (extend), `src/eval/statistics.py` (extend)

**`paired_permutation_test(a, b, n_resamples=10000, seed=12345) -> (delta, p_value)`:**
- Validates `len(a) == len(b)`; raises `ValueError` otherwise.
- Drops pairs where either side is NaN.
- Computes per-pair `diffs = b - a`; observed delta = `diffs.mean()`.
- Random sign-flips: `signs = rng.choice([-1.0, 1.0], size=(n_resamples, n))`; resample means = `(signs * diffs).mean(axis=1)`.
- Two-sided p-value: `(extreme + 1) / (n_resamples + 1)` where `extreme = sum(|resample| >= |observed|)` — Phipson & Smyth correction.

**Test cases:**
- Identical samples → p > 0.5, delta ≈ 0.
- Constant +1 shift on every pair → p < 0.01, delta ≈ 1.0.
- Same seed twice → identical output.
- Unequal lengths raise.
- NaN pairs dropped.

Commit: `feat(eval): add paired permutation test for two-run comparison`.

## Task 9 — `src/eval/metrics/refusal.py`

**Files:** `tests/test_eval_metrics_refusal.py`, `src/eval/metrics/refusal.py`

**API:**
- `is_refusal(answer: str) -> bool` — regex match against canonical refusal patterns: `cannot answer`, `unable to answer`, `does not contain`, `not (mentioned|stated|provided|specified)`, `no information`, `context does not (address|cover|contain|mention)`, `I don't know`, `cannot be answered`. Case-insensitive.
- `refusal_correctness(answer, is_unanswerable, llm) -> float` — returns 1.0 if (refused == is_unanswerable), 0.0 otherwise. Uses `is_refusal` regex first; falls back to LLM-judge with strict-JSON `{"is_refusal": bool}` schema when regex misses.

**FakeLLM test double:**
```python
class FakeLLM:
    def __init__(self, payload: dict):
        self.payload = payload
        self.calls: list[tuple[str, str | None]] = []
    def generate(self, prompt: str, system_prompt: str | None = None) -> str:
        self.calls.append((prompt, system_prompt))
        return json.dumps(self.payload)
```

**Test cases:**
- 6 parametrized canonical refusal phrasings → `is_refusal` True.
- 3 parametrized clear-answer phrasings → False.
- Unanswerable + refusal → 1.0; LLM not called.
- Unanswerable + answer → 0.0.
- Answerable + answer → 1.0.
- Answerable + refusal → 0.0.
- Ambiguous phrasing falls back to LLM judge.

Commit: `feat(eval): add refusal_correctness metric (regex + LLM fallback)`.

## Task 10 — `src/eval/metrics/generation.py`

**Files:** `tests/test_eval_metrics_generation.py`, `src/eval/metrics/generation.py`

**API:**
- `context_recall(gold_chunk_ids, retrieved_chunk_ids) -> float` — set-overlap / |gold|; NaN if gold empty. Uses FULL retrieved list (not truncated).
- `_embed(text) -> np.ndarray` — lazy global `SentenceTransformer("all-MiniLM-L6-v2")`.
- `_cosine(u, v) -> float` — defensive against zero-norm.
- `_judge_factual_match(generated, gold, llm) -> (score, reasoning)` — strict-JSON `{"factual_match": 0.0|0.5|1.0, "reasoning": str}`.
- `answer_correctness(generated, gold, llm) -> (combined, details)` — combined = mean(cosine, judge); details has both.
- Three thin wrappers around `src/evaluation.py` returning `(score, details_dict)`:
  - `judge_faithfulness(answer, contexts, llm)`
  - `judge_answer_relevancy(question, answer, llm)`
  - `judge_context_precision(question, contexts, llm)`

**Test cases:**
- `context_recall`: perfect / partial / zero / NaN-on-empty-gold.
- `answer_correctness`: high cosine + judge match → ~1.0; low + mismatch → ~0.0; partial → mean.
- Use `unittest.mock.patch` to stub `_embed` so tests don't load the SentenceTransformer.
- Three wrapper tests verify they return `(score, details_dict)` with expected keys.
